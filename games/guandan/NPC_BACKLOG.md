## 📋 后续 backlog（按优先级排序）

### P1: NPC 行为可解释性 — 已升级为 Sprint 3 后半，详见上方 milestone

### P2: B 路线 - Search-based NPC（MCTS 或浅 alpha-beta）
- 与 R 路线（规则）混合策略，先在残局阶段（≤16 张/家）使用搜索
- DoD：残局阶段胜率比纯规则路线高 ≥10%
- 预计工时：1-2 月（重大研究项）
- 风险：搜索算法的实现复杂度，可能需要专门的盘面表示

### P3: D 路线 - 监督学习初步尝试
- 用 expert 自战产生 100k+ trajectory（state, action 对）
- 训练简单 policy network（输入：手牌+局面编码，输出：出牌动作概率）
- DoD：学习到的 policy 在自战中击败 normal preset
- 预计工时：2-3 周
- 依赖：B 路线探索结果（如果 B 已能很强，D 的边际价值降低）

### P4: NPC 对真人玩家的难度调优
- 真人测试：让 N 个非家庭成员玩家分别和 expert/normal/noob 对战 5 局
- 收集"打几局后觉得无聊"和"打几局后觉得绝望"的临界点
- 调整每个 preset 的技能组合，让 normal 玩家"有挑战不绝望"
- 价值：为产品化做准备
- 预计工时：1-2 周（测试 + 调参）
- 依赖：Client Phase 3 完成（让 UX 不至于影响测试结果）

### P5: NPC 教程模式
- 让 NPC 在新手教程局里"故意"出错，引导玩家发现某些技巧
- 例：故意不让路 → 让玩家学会"队友领牌时该跟"
- 例：故意拆牌 → 让玩家学会"保留三张二组合"
- 预计工时：1-2 周（设计 + 实现教程局脚本）
- 依赖：P4（先了解人类反应，再设计教程）

## 设计原则备忘（每次推 NPC 都用得上）

1. **每个新技能必须经过消融测试再合并**——不靠"看起来更好"，靠数字。
2. **R 路线技能的"贡献量化"是核心交付物**，不是技能本身。
3. **B/D 路线启动前先把 R 路线收口（M1 达 DoD）**，否则会同时做两路混乱。
4. **NPC stream 与 Client stream 不互相依赖**，可独立推进，但不并行（WIP=1）。
